# M0 Gate 1 用户测试任务脚本（J-A / J-B）

> UAT_PLAN.md 的执行件（EPIC-M0-06 T4）。对象：光伏/建材试点画像用户（PDR-001），8–12 人。
> 方式：**任务式、无培训**——主持人只读任务卡，不解释界面；受试者边做边说（think-aloud）。
> 指标（母本 14.1 Usability）：任务完成率 / 术语理解 / 错误恢复 / 审批理解。

## 0. 会话准备（主持人，每位受试者 ~5 分钟）

1. 打开 Preview：<https://mlhjyx.github.io/global-growth-workspace/>（或本地 `http://localhost:3000`）
2. 走完登录→引导→目标向导任一路径进入工作台（或预置演示会话）
3. **洞察页 → Gate 1 验证埋点条 → 「重置采集」**（每位受试者从零开始）
4. 录屏/记录工具就位；告知受试者：测试的是产品不是你，可以随时说"我不知道这是什么"

## 1. J-A 任务脚本：主动获客（约 20 分钟）

**开场白（照读）**：「你是一家光伏组件企业的海外负责人，没有现成客户名单。请用这个系统，从设定获客目标开始，一直走到拿到一个销售认可的商业机会。过程中你觉得该点什么就点什么。」

| # | 任务卡（照读给受试者） | 成功判定（主持人观察） | 采集 |
|---|---|---|---|
| A1 | 「启动一个找客户的目标」 | 从今日页目标启动器点「找客户」，顶部出现旅程导航条 | 完成/求助/放弃；耗时 |
| A2 | 「了解你要进入的市场」 | 在研究页浏览扫描/研究结果后，点导航条「完成本步」 | 同上 + 是否点开任何证据 |
| A3 | 「确认系统建议的理想客户画像」 | 在客户页理解 ICP 与六维评分含义并推进 | **术语探针**：请受试者用自己的话说"ICP/六维评分是什么" |
| A4 | 「挑出一批你认可的合格线索」 | 在四队列中查看 Score Explain 并推进 | 是否打开 Score Explain/依据 |
| A5 | 「把这批线索变成一个可执行的战役」 | 在战役页查看画布（受众/内容/序列）并推进 | 完成/求助/放弃 |
| A6 | 「让系统获得对外发送的许可」 | 在授权区完成 Dry-Run 查看与「批准并签发授权」 | **审批探针**：请受试者说"你刚才批准的是什么、系统现在能做什么/不能做什么" |
| A7 | 「处理客户的回复」 | 在互动收件箱查看回复上下文并确认意向 | 完成/求助/放弃 |
| A8 | 「确认这个机会已被销售接受」 | 在洞察页结果链找到 QL→SAO 并说出含义 | **术语探针**：SAO 用自己的话解释 |

**结束判定**：旅程导航条显示「旅程走通：目标 → SAO 全链完成」。

## 2. J-B 任务脚本：经销商招募（约 15 分钟）

**开场白（照读）**：「换一个身份：你是建材（石膏板）厂家，要在尼日利亚找经销商。从设定目标开始走到确认一个合格渠道伙伴机会。」

| # | 任务卡 | 成功判定 | 采集 |
|---|---|---|---|
| B1 | 「启动找经销商的目标」 | 目标启动器「找经销商」→ 旅程导航条切换为 J-B | 完成/求助/放弃 |
| B2 | 「看看这个市场的准入要点」 | 研究页市场/行业包内容浏览后推进 | 同上 |
| B3 | 「找到候选经销商名单」 | 客户页查看渠道线索并推进 | 同上 |
| B4 | 「准备给经销商看的材料」 | 内容页查看基于 Approved Claim 的内容并推进 | **术语探针**：Approved Claim 是什么 |
| B5 | 「让专家把关本地化口径」 | 专家页（占位）理解其用途并推进 | 是否理解"占位=未来能力" |
| B6 | 「授权系统发出招募邮件」 | 战役页授权区批准 | **审批探针**（同 A6） |
| B7 | 「处理经销商的回复」 | 互动页确认合作意向并推进 | 完成/求助/放弃 |
| B8 | 「确认这个渠道机会」 | 洞察页结果链推进至完成 | 完成/求助/放弃 |

## 3. 错误恢复探针（嵌在任一旅程中执行一次）

主持人指引：「打开发布页/互动页里带『部分成功/失败』标记的区块，处理一下失败的那几条。」
**成功判定**：受试者自行找到「单独重试」或恢复动作并点击（埋点 error_recover 自动计数）。

## 5. 判定阈值（进 Gate 1 Report）

- 任务完成率（无培训、无求助完成的任务占比）≥ **70%**
- 术语理解：四项探针平均 ≥ **1.2 / 2**
- 错误恢复：自主完成率 ≥ **60%**
- 审批理解：得分 ≥ **1.5 / 2**（这是硬边界 1 的用户侧验证——批准≠系统失控）

阈值未达 → 按 docs/testing/GATE1_REPORT_TEMPLATE.md 记录差距与整改项，Gate 1 判定 CONDITIONAL/FAIL。
